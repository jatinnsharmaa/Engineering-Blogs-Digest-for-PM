# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Running the pipeline

```bash
# activate venv first
source ../pm-digest-env/bin/activate

# test run — generates digest_preview.html, no email sent
python main.py --dry-run

# live run — fetches, summarises, compiles, emails
python main.py

# print the cron expression after changing schedule in settings.yaml
python make_cron.py
```

`main.py` must be run from the repo root (not from inside `agents/`), because agents load prompts via relative paths like `prompts/summarizer.txt`.

Required env var: `ANTHROPIC_API_KEY` (set in `.env` or exported). Gmail credentials must exist at `auth/credentials.json` and `auth/token.json` (generated by `python auth/setup.py`).

## Architecture

The pipeline is a sequential four-agent workflow orchestrated by `main.py`:

```
FetcherAgent → List[Article] → SummarizerAgent → List[ArticleSummary] → CompilerAgent → DigestContent → SenderAgent
```

**FetcherAgent** (`agents/fetcher.py`) has two modes unified into one `run()` call:
- **Gmail mode** (`type: gmail` in sources.yaml): queries Gmail for unread emails from whitelisted sender addresses. Stores Gmail message IDs in `_pending_gmail_ids`; these are marked as read only after a successful send via `mark_gmail_processed()`.
- **RSS mode** (`type: rss`): polls Atom/RSS feeds with `feedparser`, extracts article body with `trafilatura`. Uses SQLite (`db/digest.db`) to track processed GUIDs; committed via `mark_rss_processed()` only after a successful send.

Sources with `enabled: false` are skipped before fetching. Both modes return `Article` dataclasses. State is never committed until the email send succeeds.

**SummarizerAgent** (`agents/summarizer.py`) calls `claude-haiku-4-5-20251001` once per article with prompt caching (`betas=["prompt-caching-2024-07-31"]`, `cache_control: ephemeral` on the system prompt). Returns `None` for articles where Claude sets `"skip": true` (not PM-relevant). Articles are batched in groups of 5 with a 1s sleep between batches.

**CompilerAgent** (`agents/compiler.py`) makes a single `claude-sonnet-4-6` call with all summaries, grouping them into 3–5 themes and writing an editorial intro/closing. Returns `DigestContent`. If Claude returns malformed JSON, raises `ValueError` with the raw output.

**SenderAgent** (`agents/sender.py`) renders `templates/digest.html.jinja2` and sends via Gmail API. Raises on failure — never silently succeeds.

If no articles are found or all are skipped, `main.py` sends a "quiet week" email rather than exiting silently (`_empty_digest()`).

## Configuration

**`config/sources.yaml`** — the only file that needs regular editing. Fields per source:
- `enabled: true/false` — set to `false` to pause a source without deleting it
- `type: rss` with `url:` to the feed
- `type: gmail` with `sender:` email address and optional `subject_contains:` for noise filtering

**`config/settings.yaml`** — models, batch sizes, lookback window (default 7 days), db path, email addresses, and schedule:
```yaml
schedule:
  day: Saturday   # Mon/Tue/Wed/Thu/Fri/Sat/Sun
  hour: 9         # 24-hour UTC
```
After changing the schedule, run `python make_cron.py` and paste the output into `.github/workflows/weekly_digest.yml`.

**`prompts/summarizer.txt`** and **`prompts/compiler.txt`** — the full LLM system prompts. Edit these to change framing, output fields, or tone. The summarizer prompt defines the JSON schema that `ArticleSummary` depends on.

## Data flow and state

- Gmail state: read/unread status in Gmail itself
- RSS state: `db/digest.db` (SQLite, single table `rss_articles` with columns `guid, company, url, processed_at`)
- In CI, `db/digest.db` is persisted across weekly runs via GitHub Actions artifacts (90-day retention), not cache

## Adding a new source

In `config/sources.yaml`:
```yaml
- name: Company Engineering
  enabled: true
  type: rss
  url: https://example.com/feed
```
For `type: gmail`, also set `sender:` to the exact From address of their newsletter emails. Use `subject_contains:` if the sender address is shared (e.g. `noreply@medium.com`).
